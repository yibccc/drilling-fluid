## 2. 服务概况

### 2.1 基础信息

| 项目 | 值 |
|------|----|
| 默认地址 | `http://localhost:8000` |
| 协议 | HTTP |
| 返回格式 | JSON / SSE |
| 健康检查 | `GET /health` |

### 2.2 认证方式

除 `/health` 外，其余接口都通过 `get_user_id()` 鉴权。

支持两种 Header：

| Header | 用途 |
|--------|------|
| `X-API-Key` | 正常用户调用 |
| `X-Internal-Api-Key` | 内部系统调用，命中 `INTERNAL_API_KEY` 时识别为内部请求 |

未提供有效认证时返回：

```json
{
  "detail": "Missing authentication"
}
```

### 3.3 AI 诊断分析

**POST** `/api/v1/diagnosis/analyze`

用途：

- 接收 `DiagnosisRequest`
- 调用 `DiagnosisService.analyze()`
- 以 `StreamingResponse` 返回 `text/event-stream`

请求体字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `alert_id` | string | 是 | 预警 ID |
| `task_id` | string | 否 | 不传时由服务端模型生成 |
| `well_id` | string | 是 | 井号 |
| `alert_type` | string | 是 | 预警类型 |
| `alert_triggered_at` | datetime | 是 | 触发时间 |
| `alert_threshold` | object | 是 | 预警阈值 |
| `samples` | array | 是 | 1 到 20 条采样数据 |
| `context` | object | 否 | 诊断上下文 |
| `stream` | boolean | 否 | 默认 `true` |

`alert_threshold` 字段结构：

```json
{
  "field": "density",
  "condition": "greater_than",
  "threshold": 1.30,
  "current_value": 1.35
}
```

`samples` 中每条采样记录当前要求包含以下字段：

- `id`
- `well_id`
- `sample_time`
- `formation`
- `outlet_temp`
- `density`
- `gel_10s`
- `gel_10m`
- `rpm_3`
- `rpm_6`
- `rpm_100`
- `rpm_200`
- `rpm_300`
- `rpm_600`
- `plastic_viscosity`
- `yield_point`
- `flow_behavior_index`
- `consistency_coefficient`
- `apparent_viscosity`
- `yield_plastic_ratio`

当前实现实际输出的 SSE 事件以这几类为主：

| `type` | 说明 |
|--------|------|
| `start` | 开始分析 |
| `thinking` | 中间过程，`step` 常见为 `data_analysis`、`analyzing`、`reasoning`、`tool_call`、`tool_result` |
| `result` | 最终结构化诊断结果 |
| `done` | 完成事件 |
| `error` | 错误事件 |

成功响应示例：

```http
data: {"type":"start","task_id":"TASK-001","well_id":"WELL-TEST","content":"开始分析井号 WELL-TEST 的 1 条采样数据"}

data: {"type":"thinking","task_id":"TASK-001","content":"正在分析数据并生成诊断结果...","step":"analyzing"}

data: {"type":"result","task_id":"TASK-001","result":{"diagnosis":{"summary":"...","cause":"...","risk_level":"MEDIUM"},"measures":[],"prescription":{}}}

data: {"type":"done","task_id":"TASK-001","status":"SUCCESS"}
```

常见错误：

| 场景 | HTTP / 事件 |
|------|-------------|
| 服务未初始化 | `503` |
| 请求体验证失败 | `422` |
| 流式执行中抛出业务异常 | SSE `error` 事件 |

### 3.4 查询诊断结果

**GET** `/api/v1/diagnosis/{task_id}`

用途：

- 查询任务记录
- 查询任务结果
- 组合返回 `{ task, result }`

成功响应示例：

```json
{
  "task": {
    "task_id": "TASK-001",
    "well_id": "WELL-TEST",
    "status": "SUCCESS"
  },
  "result": {
    "task_id": "TASK-001",
    "diagnosis": {
      "summary": "密度上升",
      "risk_level": "MEDIUM"
    }
  }
}
```

任务不存在时返回 `404`：

```json
{
  "detail": "Task not found"
}
```

### 3.5 知识召回调试

**POST** `/api/v1/diagnosis/knowledge/search`

请求体：

```json
{
  "query": "密度偏高怎么办",
  "doc_id": "DOC-001",
  "category": "density",
  "top_k": 3
}
```

说明：

- `doc_id`、`category` 都是可选过滤条件
- 路由内部固定追加 `chunk_type=child`，只返回子分块检索结果

成功响应示例：

```json
{
  "query": "密度偏高怎么办",
  "doc_id": "DOC-001",
  "category": "density",
  "top_k": 3,
  "results": [
    {
      "doc_id": "DOC-001",
      "title": "密度处理指南",
      "category": "density",
      "chunk_id": "DOC-001_parent_child_0",
      "parent_chunk_id": "DOC-001_parent",
      "chunk_type": "child",
      "content_preview": "密度偏高时，应加水稀释"
    }
  ]
}
```

### 3.6 删除知识文档分块

**DELETE** `/api/v1/diagnosis/knowledge/documents/{doc_id}`

用途：

- 查出该 `doc_id` 对应的全部向量分块
- 按 `chunk_id` 删除
- 返回删除数量

响应示例：

```json
{
  "doc_id": "DOC-001",
  "deleted_count": 4,
  "status": "deleted"
}
```
